# Release Notes — Phase 2.9 (Evidence Runtime)

ContentCompiler v2.9.0 completes the Evidence Runtime. Delivery outcomes can now enter the architecture as normalized, immutable observations anchored to their exact delivery artifact.

## Highlights

- Platform metrics become typed PerformanceSignals with a required observation time.
- Human and operational feedback becomes typed HistoricalObservations.
- Evidence aggregates preserve a single, traceable delivery lineage and cannot silently change Knowledge or strategic state.
- The runtime validates all incoming observations before persistence.

Phase 2.10 — Integration: connect the eight runtime layers through a deterministic end-to-end flow using mock providers.
